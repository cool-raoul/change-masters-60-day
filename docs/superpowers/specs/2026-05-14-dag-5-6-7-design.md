# Dag 5, 6 en 7 — Design

**Doel:** Dag 5, 6 en 7 van het Sprint-playbook tempo-aware maken in dezelfde stijl en flow als dag 3+4, met opfriste content (geen externe namen in de dag-flow), consistente sponsor-checkin-als-afsluiter, en een nieuwe Audio-onderweg-Academy-training waar Eric Worre wel bij name mag.

**Status:** Brainstorm-akkoord 2026-05-14 (Raoul). Klaar voor implementatie-plan.

---

## 1. Achtergrond

Dag 1 t/m 4 zijn al uitgewerkt:

- **Dag 1**: 5 namen handmatig + vCard-import + sponsor-melding
- **Dag 2**: 20 namen verbreden + Mentor-stem + sponsor-call
- **Dag 3**: Tempo-aware ritme (A namen, B berichten, follow-ups, stories, teams-admin, sponsor-checkin als laatste)
- **Dag 4**: Tempo-aware met 4-stappen-uitnodiging als hoofd-stap

Dag 5, 6 en 7 staan al in [lib/playbook/dagen.ts](../../../lib/playbook/dagen.ts) met basis-content maar:
- Niet tempo-aware (vaste aantallen 5/3/3)
- Bevatten externe trainer-namen (Eric Worre, Frazer Brookes, Spotify-podcast-link)
- Sponsor-stappen staan niet consistent als laatste
- "3 namen uit socials" als aparte stap, terwijl dag 3+4 dat al via 4-vlakken-blok dekken

Dit ontwerp brengt dag 5-7 op gelijke voet met dag 3+4 en bouwt voort op:
- De [DMO-training](../../../lib/academy/dmo-content.ts) ("Het Dagelijkse Ritme") die wiskunde van drop-off + wet-van-80% al uitlegt
- De [Spreken-zoals-het-raakt-training](../../../lib/academy/claim-vrij-content.ts) die toon-consistentie vraagt
- Het [Spreken-zoals-het-raakt brondocument](../../claimvrije-communicatie.md) voor claim-vrije taal

## 2. Architectuur

### Bestaande pattern (van dag 3+4)

```
DAGEN[dagN] (statische basis in dagen.ts)
    ↓
pasTempoToeOpDag(dag, commitmentUren)  in tempo-aware.ts
    ↓
bouwDagNVandaagDoen(uren)              returnt ControllableTaak[]
    ↓
override-passes (founder-edits via tekst_overrides)
    ↓
vandaag-flow.tsx rendert
```

Dit blijft hetzelfde voor dag 5-7. Alleen `pasTempoToeOpDag` wordt uitgebreid met cases voor 5, 6 en 7, en er komen drie nieuwe builders bij.

### Nieuwe helper: `dagdoelenMinimum`

Naast de bestaande [`berekenDagdoelen`](../../../lib/dagdoelen.ts) komt een helper voor dag 7 minimum-aantallen.

```ts
export type DagdoelenMinimum = {
  contacten: number;       // 3 / 5 / 8 per tempo
  uitnodigingen: number;   // 1 / 2 / 3
  followups: number;       // 1 / 3 / 5
};

export function dagdoelenMinimum(uren: CommitmentUren): DagdoelenMinimum {
  switch (uren) {
    case 2: return { contacten: 3, uitnodigingen: 1, followups: 1 };
    case 4: return { contacten: 5, uitnodigingen: 2, followups: 3 };
    case 6: return { contacten: 8, uitnodigingen: 3, followups: 5 };
  }
}
```

Filosofie: niet halveren maar ondergrens — "minimaal X, meer mag, minder niet".

### Nieuwe Academy-training: Audio onderweg

Hergebruik van bestaande `AcademyTraining`-structuur uit [lib/academy/types.ts](../../../lib/academy/types.ts), met één optionele veld-uitbreiding:

```ts
export type AcademyLes = {
  // ...bestaande velden...
  /**
   * Optionele Spotify-zoeklink. Wanneer gezet, rendert de les-pagina
   * een knop "Beluister op Spotify" naast de leestekst.
   */
  audioZoekLink?: string;
};
```

De nieuwe training heeft één module met 8 "lessen", waarvan les 0 een intro is met de hint "luister deze eerst" en lessen 1-7 de Seven Skills in vrije volgorde mogen.

Op dag 6 krijgt deze stap een UITGEBREIDE uitleg variant — niet `FOLLOWUP_UITLEG_BASIS` maar een eigen `FOLLOWUP_UITLEG_DAG6` constant in tempo-aware.ts die:

- Begint met de 24-48u-regel (eerste check-in 24-48u na uitnodiging)

export function pasTempoToeOpDag(dag: Dag, commitmentUren: CommitmentUren | null): Dag {
  if (commitmentUren === null) return dag;

  if (dag.nummer === 4) return { ...dag, vandaagDoen: bouwDag4VandaagDoen(commitmentUren) };

## 10. Data-flow en error handling

### Geen DB-migraties nodig

Alle aanpassingen zijn content + UI. Geen schema-veranderingen.

### Founder-overrides blijven werken

De tempo-aware-functies returnen `ControllableTaak[]`. Override-passes worden bovenop deze taken gelegd in de vandaag-flow zoals nu — geen wijziging aan dat mechanisme nodig.

### Backwards compatible

`dagdoelenMinimum` is een NIEUWE export. `audioZoekLink` is een OPTIONEEL veld op `AcademyLes`. Bestaande trainingen die het niet gebruiken blijven ongewijzigd werken.

### Voortgang in DB

De `training_voortgang`-tabel werkt al met `training_slug` + `les_sleutel` als unieke key. Voor `audio-onderweg` wordt automatisch een nieuwe slug-rij aangemaakt zodra een member de eerste track-les markeert. Geen migratie nodig.
